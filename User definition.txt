User is defined and used as follows:

-- USERNAME --
Used for login. Selectable by the user themselves.

-- PASSWORD (hash) --
Used for login. Hashed upon entry and sent to the database where it is used for validation.
REMEMBER THAT THE PASSWORD ITSELF IS NEVER SENT ANYWHERE.

-- IDENTIFICATION NUMBER --
Used for Database structure. Some uncertainty as to exactly how.
Automatically generated by the system (see: hashes?). Constant.

-- OWNS --
Database list entry that determines which presentations this user has ownership of.
Means: private, shared and public presentations you can freely edit, un/share and delete.

-- ACCESS --
Database list entry that determines which presentations this user has access to.
Means: shared or public presentations are included. Cannot DELETE these. Cannot SHARE these. Can UNSHARE these.
